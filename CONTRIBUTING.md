# Посібник для контрибʼюторів

## Робочий процес

1. Форкніть репозиторій та створіть гілку від `main`.
2. Створіть зміни невеликими ітераціями, дотримуйтесь атомарних комітів.
3. Перед відкриттям Pull Request перевірте форматування та лінти.
4. Описуйте зміни у PR відповідно до шаблону (що зроблено, як тестувалося, релевантні питання).

## Стиль коду

- TypeScript strict mode, без `any`.
- Дотримуйтесь принципів SOLID та DRY.
- Використовуйте React Server Components, де це доречно, та розмежовуйте клієнтські компоненти директивою `'use client'`.
- Для форм — React Hook Form + Zod. Усі вхідні дані мають бути валідовані на сервері.
- Tailwind: використовуйте `cn` та `buttonVariants` для композиції класів.

## Команди якості

```bash
pnpm lint
pnpm typecheck
pnpm test
```

## Git-конвенції

- Формат комітів: `feat: ...`, `fix: ...`, `chore: ...`, `docs: ...`, `refactor: ...`.
- Один Pull Request — одна фіча/виправлення.
